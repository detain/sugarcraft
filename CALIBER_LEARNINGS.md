# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[pattern]** Binary test fixtures (raw ANSI/control byte streams under `tests/fixtures/*.ansi`) need a dir-local `.gitattributes` declaring `*.ansi binary` — without it, git's `core.autocrlf` may rewrite `\r\n` ↔ `\n` on checkout and silently corrupt CR/LF/ESC byte sequences in the fixture. Generate the bytes via `printf '\x1b[31mR\x1b[0m' > tests/fixtures/foo.ansi` rather than an editor (which may insert/strip BOMs or normalise newlines). Verify with `bin2hex(file_get_contents(...))` from a `php -r` one-liner using an absolute path.
- **[pattern]** When auditing whether a newly-added library is FULLY integrated, the easy-to-miss artifacts are (a) `media/icons/<slug>.png` AND `docs/img/icons/<slug>.png` (BOTH paths — `media/` is the source-of-truth icon, `docs/img/icons/` is what the website serves), (b) the `docs/index.html` homepage tile in the `.logo-strip` section (`<a href="lib/<slug>.html" title="<Brand> — <one-liner>"><img src="img/icons/<slug>.png" alt="<Brand>"></a>`), (c) the `docs/lib/<slug>.html` detail page (use `docs/lib/candy-core.html` as the reference layout — head/og meta, nav, `.detail-header` with icon + h2 + tag-chips, install/quickstart/feature-grid/source/demos sections, footer), (d) `<slug>/examples/*.php` runnable demos, (e) `<slug>/.vhs/*.tape` + matrix entry in `.github/workflows/vhs.yml`. Nothing fails without them — composer/CI/tests all pass — so they linger. Audit checklist: `ls media/icons/<slug>.png docs/img/icons/<slug>.png docs/lib/<slug>.html <slug>/examples <slug>/.vhs && grep -q '<slug>' docs/index.html .github/workflows/vhs.yml`.
- **[pattern]** When a new lib needs a placeholder icon, two interchangeable approaches: (a) ImageMagick `convert -size 256x256 xc:transparent -fill '<hex>' -draw "..."  <path>.png`, or (b) PHP GD via `php -r '$im = imagecreatetruecolor(256, 256); ...; imagepng($im, $path);'` (works without ImageMagick). Write to BOTH `media/icons/<slug>.png` and `docs/img/icons/<slug>.png` in the same script. Verify with `file <path>` — should report `PNG image data, 256 x 256`. Existing icons are 240² but the contributor playbook calls for 256² — both render fine in `docs/index.html`'s `.logo-strip` (CSS resizes), so don't waste time matching the legacy 240² dimension.
- **[gotcha]** After `gh pr create` / `gh pr merge --delete-branch`, the script prints `Warning: N uncommitted changes` (any small N) even on a clean tree — this is `gh` complaining about untracked files like `.claude/scheduled_tasks.lock` or sibling `plans/` files left over from another branch's work, not a real uncommitted-source warning. Don't react to it; verify with `git status` if unsure.
- **[pattern]** The `plans/x-<feature>.md` file's `## Tracking` section enumerates the post-implementation sweep — files outside the implementation PRs that need updates after the LAST PR merges (UPSTREAM_OPPORTUNITIES status icon, root CALIBER_LEARNINGS gotchas, README/AGENTS platform requirements, `docs/index.html` tile, `docs/lib/<slug>.html` detail page, `media/icons/` + `docs/img/icons/` icons, vhs.yml matrix entry, examples/ + .vhs/.tape demos, `.claude/rules/<topic>.md` if conventions changed). Treat this as a checklist; do a final tracking-only PR (e.g. `ai/<feature>-pr9-finish`) when all implementation PRs land, since these doc updates don't fit into any individual implementation PR's scope.
- **[gotcha]** The Bash tool's working directory persists across separate Bash calls. After `cd /home/sites/sugarcraft/<lib> && cmd`, the NEXT Bash call still starts in `<lib>` — `git status` will then show paths with `../` prefixes (e.g. `modified: ../docs/index.html`) that look like they're outside the repo but aren't. Always use absolute paths or prefix every Bash call with `cd /home/sites/sugarcraft && ...` to anchor it; if you get a mysterious empty response from `ls`/`cat` or a `cat: .github/workflows/vhs.yml: No such file or directory` error on a file that obviously exists, suspect the cwd before suspecting the file. Same trap bites `php -r "file_get_contents(__DIR__.'/tests/fixtures/foo')"` — `__DIR__` in `-r` mode resolves to the cwd at exec time. Use absolute paths in `php -r` snippets.
- **[pattern]** Project-specific agent behavior rules live in `.claude/rules/<topic>.md` with frontmatter `paths: ['*/tests/**/*.php']` (or similar glob), NOT in CLAUDE.md/AGENTS.md. These get auto-loaded for matching files. When project-wide test conventions evolve (e.g. recommending `candy-vt`-driven cell-grid snapshots over raw-byte snapshots for downstream libs), edit `.claude/rules/test-conventions.md` rather than CLAUDE.md — keeps the main playbook stable and the rule scoped to where it applies.
- **[pattern]** Two snapshot test flavors, both valid: *byte snapshot* (assert raw `\x1b[1m...` bytes from `view()`) for renderer-internal tests where you're verifying the ANSI output itself; *cell-grid snapshot* (drive bytes through `SugarCraft\Vt\Terminal\Terminal`, then assert `$term->screen()->cell($r, $c)->grapheme/sgr/foreground()` and `$term->cursor()`/`$term->mode()`) for downstream/integration tests. The cell-grid form survives ANSI-level reordering — redundant SGR re-emission, equivalent cursor moves — that doesn't change what the user sees. See `candy-vt/tests/SnapshotTest.php` for fixture-driven examples.
- **[pattern]** When writing tests that drive a VT parser/handler with raw byte sequences, set up buffer state with explicit CSI cursor-position moves (`\x1b[r;cHTEXT`) rather than chains of print + CR (`\x0D`) + LF (`\x0A`) + overwrite. Mental-tracing the latter is error-prone — rewriting them as `\x1b[1;1HAB\x1b[2;1HCD\x0A` (explicit row,col placement, then exactly one LF to trigger the behavior under test) made the assertions trivially correct. Reserve C0/CR/LF chains for tests that are *specifically* about CR/LF semantics.

- **[gotcha]** `end()`, `reset()`, `next()`, `prev()`, `array_pop()`, `array_shift()`, `sort()`, etc. all take their argument by reference and CANNOT be called on a `public readonly array $foo` property — you get `Error: Cannot modify readonly property` pointing at the line that calls the function (not at any assignment), which is misleading. Use index access (`$this->foo[count($this->foo) - 1]`) or `array_key_last($this->foo)` instead. This bites both `final readonly class` properties and per-property `public readonly array` declarations equally — the readonly modifier blocks the by-ref pointer mutation that those funcs perform.
- **[gotcha]** When a new lib subdir is added (e.g. `candy-vcr/`), the caliber pre-commit hook does NOT auto-stage untracked files in the new dir — `git add -A` would, but per project convention you stage explicit paths to avoid sweeping in `.env`-class files. Stage every new file explicitly: `git add candy-<slug>/composer.json candy-<slug>/phpunit.xml candy-<slug>/README.md candy-<slug>/CALIBER_LEARNINGS.md candy-<slug>/src/...` plus the cross-cut edits (`composer.json` root, `.github/workflows/ci.yml`, `MATCHUPS.md`, `PROJECT_NAMES.md`, `CONVERSION.md`, `UPSTREAM_OPPORTUNITIES.md`). `git status -uall <subdir>/` enumerates the per-file list when you forget what you created.
- **[pattern]** When wiring a new lib into the root `composer.json`, BOTH the `require` block (add `"sugarcraft/<slug>": "@dev"`) AND the `repositories` block (add `{"type":"path","url":"<slug>","options":{"symlink":true}}`) need entries — missing the repositories block produces `Could not find package` at install time even though the directory exists alongside. Pair the two edits in the same step.
- **[pattern]** When swapping a stub method (e.g. `Terminal::feed()` no-op → real Parser+Handler pipeline) for a working implementation, audit the test file FIRST and replace the corresponding stub-era test (`testFeedIsNoOpInPr1`, `testPrintsMultiByteUtf8AsSingleCell`, etc.) before running the suite — leaving the stub-era assertion in place produces a confusing failure that looks like the new wiring is broken ("expected 1 got 2" cell count for CJK after Width wiring, or "expected ' ' got 'R'" for Parser+Handler) when it's actually the test that's outdated. PHPUnit's diff output shows the new behavior on the `+` side, which is a hint to update the test rather than the code.

Use `--json state,mergedAt` instead: a merged PR returns `{"state":"MERGED","mergedAt":"<timestamp>"}`, an open PR returns `"mergedAt":null`. - **[gotcha]** `candy-core`'s `Program::run()` automatically dispatches three startup Msgs to `model->update()` BEFORE any caller-sent Msg, in this order: `WindowSizeMsg`, `EnvMsg`, `ColorProfileMsg`. So a test model that uses an `int $count`/`quitAfter` to issue `Cmd::quit()` after N updates needs to account for the 3 baked-in ticks. With a single `$program->send(new KeyMsg(...))` and a 2.0s loop-stop timer, a `quitAfter: 5` model only fires 4 updates (3 startup + 1 sent KeyMsg) and never reaches the quit branch — `QuitMsg should produce a quit event` then fails with no clear root cause in PHPUnit output. Use `quitAfter: 4` for that exact shape, or quit on a `KeyMsg` predicate instead of a counter when testing recorder/lifecycle plumbing.
- **[gotcha]** Streaming-writer classes whose `close()` method calls `fclose($this->fh)` (e.g. `candy-vcr`'s `Recorder::close()`) make the underlying stream resource invalid — calling `rewind($fh)` afterwards in the same test throws `TypeError: rewind(): supplied resource is not a valid stream resource`. Test pattern: do NOT call `$r->close()` if you also want to `rewind`/`fgets`/`stream_get_contents` the same `php://memory` handle to inspect the bytes. Either skip the close in tests that need to read back, or close once and then re-open via the file path (when using `Recorder::open($path)` rather than the in-memory ctor).
- **[pattern]** When PHPUnit failure messages alone don't explain a Program/Recorder/lifecycle test failure (assertion phrasing like "Failed asserting that an array contains EventKind Enum #X (Quit, 'quit')" doesn't tell you which Msgs DID flow), stage a standalone debug script at `/tmp/dbg.php` with `echo "tick $count: " . get_class($msg)` inside `update()`, run with absolute-path autoload, and inspect the printed Msg sequence. Faster than reading the 800-line Program.php dispatch. Remember to use `require_once '/home/sites/sugarcraft/<lib>/vendor/autoload.php'` (absolute), not `__DIR__ . '/vendor/autoload.php'`.
- **[gotcha]** `candy-core`'s `setupTerminal()` emits output bytes (e.g. `\x1b[?2027h` unicodeOn, alt-screen-enter, cursor-hide) BEFORE the startup `WindowSizeMsg` dispatch. Anything teeing `Program::writeOutput()` (recorder, debug tap, network mirror) sees those bytes as the FIRST event(s); the `resize` event arrives 1–3 events in. Tests asserting cassette/tap shape must scan for `Resize` rather than indexing `events[0]`. Same applies for the tail: `teardownTerminal()` emits `\x1b[?2027l` etc. AFTER `dispatch(QuitMsg)`, so the cassette would end on environmental noise unless the recorder is closed inside the `QuitMsg` branch.
- **[pattern]** Tee/sink classes attached to `Program` (recorder, debug tap, mirror) should make every `record*`/`write*` method **no-op after close()**, NOT throw `LogicException`. Reason: the runtime has `dispatch(QuitMsg) → close → teardownTerminal → writeOutput(...) → sink->record...` ordering — teardown's environmental writes (unicodeOff, alt-screen-leave, cursor-show) can't be intercepted by the caller, so a throw-on-closed sink would crash on every clean shutdown. Inverse: `Program::dispatch(QuitMsg)` must call `recorder?->recordQuit(); recorder?->close();` immediately (not at end of `run()`), so the cassette ends on a clean `quit` line and the no-op-after-close drops the teardown noise.
- **[pattern]** Cross-lib hook seams that need to live in candy-core but are implemented in a downstream lib (e.g. recorder lives in candy-vcr but the tee point is `Program::writeOutput()` in candy-core) take the form: candy-core ships an INTERFACE (`SugarCraft\Core\Recorder` with `recordResize/recordInputBytes/recordOutput/recordQuit/close`), `Program::withRecorder(?Recorder $r): self` accepts the interface, and the downstream lib provides the concrete class implementing it. Avoids candy-core depending on candy-vcr (the dependency goes the other way) and lets other consumers (debug tap, network mirror, crash bundle) slot into the same hook without pulling cassette plumbing.
